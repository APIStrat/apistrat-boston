API design is one of the most important aspects of the API lifecycle in 2015. It is one of the fastest growing areas, reflects the most pressing issues that face API providers, and is poised to lead almost every other stop along the API lifecycle. When you think about API design, think the drummer in a rock band--API design will set the beat for everything  that happens, at every stage of an APIs life.

Your API design decision have the potential to affect every other aspect of API operations from testing and monitoring, to support and documentation. The best way to minimize mistakes along the way, and maximize API design success, is to learn from leading API design experts across the space, at intimate API focused events like API Days / APIStrat in Berlin this spring.

Weve invited three experts from different areas of the API space:

  • Orlando Kalossakas (@orliesaurus) of Mashape - Designing and building APIs, creating great documentation, and mocking
  • David Garry (@davidgarry6) of Nomos Software - Extending Swagger for Virtual and Diagnostic Rich APIs
  • Jason Harmon (@jharmn) from Paypal - Technology/Design or Best Practice (issues learned from running the Paypal program)

We still have one slot available in our API design session, this doesn't mean we need to find someone, this means we have a couple of options, and are having a hard time deciding what would be the cherry on top of the API design discussion in Germany. Orlando, David, and J(a)son are bringing some serious experience to the table, and the fourth slot should bring it home.

When it comes to any aspect of the API economy, API design is the most critical layer that the entire API community should be working to share expertise, best practices, and stories from the trenches. If you are unsure of what the best practices are, you should be tuning into the conversation that happen at API Days / APIStrat.

Make sure and get registered for the conference, and we will see you in Berlin, April 24th and 25th, 2015.
